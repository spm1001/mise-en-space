google-auth's `Credentials.valid` property only checks the local `expiry` field. If `expiry` is None (which happens with some token file formats from jeton), it reports `valid=True` even when the token has expired server-side. The old `AuthorizedHttp` from google-auth-httplib2 handled this transparently by retrying on 401. Any replacement HTTP client must do the same — retry once on 401 with a forced `credentials.refresh()`. This bit us during the httpx migration and would have been invisible without integration tests. Unit tests that mock the HTTP client can never catch this class of bug.
