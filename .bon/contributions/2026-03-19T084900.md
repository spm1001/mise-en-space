Plugin token storage must not live in the versioned cache dir — it dies on version bump. The `~/.claude/plugins/data/{name}-{publisher}/` directory is version-stable and created by CC automatically, but undocumented. We adopted it with a fallback chain (data dir → legacy package root) so the code degrades gracefully if CC changes the convention. The token_store.py `resolve_token_path` function auto-migrates from old locations on first access. Created files now carry provenance (`description` + `properties.mise=true`) so ghost docs from Claude using placeholder titles are identifiable and searchable via Drive API (`properties has { key='mise' and value='true' }`).
