## For Claudes to come

**The shadow fields mask.** When an API adapter defines a constant for field selection but the actual call site uses a locally-defined string, tests that mock API responses will pass even when the real mask is wrong — mocks return whatever you tell them regardless of what fields were requested. The `is_unread` bug was exactly this: `SEARCH_THREAD_FIELDS` had `labelIds`, but `search_threads()` used a local `search_fields` string that didn't. All tests passed because the mock responses included `labelIds` in the data. The guard is testing the mask itself (`assert "labelIds" in fields`) rather than just the downstream logic. Any time you see a fields/projection/select mask in an API adapter, check whether the actual call site uses the constant or a shadow copy.
