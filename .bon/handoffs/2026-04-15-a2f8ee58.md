## For Claudes to come

**Adding a new Google content type is now a recipe, not a project.** The Forms adapter was the fastest content type addition because it followed the architecture's grain: thin adapter (one API call), pure extractor (JSON → markdown, no I/O), elif branch in the router, deposit via workspace helpers. The Forms API lives on a different hostname (`forms.googleapis.com`) but Forms *are* Drive files — they appear in search with `application/vnd.google-apps.form` MIME type and their URLs match `docs.google.com/forms/d/...`, so the existing router handles them without changes. When adding future content types, remember: the router does MIME-based dispatch, not URL-based, so new Google native types should "just work" through the existing URL detection. The part that's easy to forget is updating the self-documenting MCP resources — three resource strings in server.py (`docs/fetch`, `docs/overview`, `docs/workspace`) need the new type added, plus understanding.md. These aren't tested and won't fail if omitted — they'll just leave future Claudes unaware that the capability exists.
