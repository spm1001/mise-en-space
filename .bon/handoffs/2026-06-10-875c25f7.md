## For Claudes to come

A green check that doesn't exercise the real failure is the same bug at every layer, and 2026-06-10 produced the full set: a unit test whose comment described the contract ("not me (self)") but never asserted it; a deployment guard that verified plugin.json exists but not that its promises resolve (the Cowork husk — every session got a dead server while four guards stayed green); a verification probe that ran an empty file and exit-0'd vacuously. Three correctives fired repeatedly and all worked: **demand the failure first** — run a new check against known-bad before trusting any green from it, because a check that has never failed has never been tested; **verify the product, not the proxy** — read the bot's commit contents, not the green run badge; **when fixing an absence, inventory-diff the presences** — the husk fix restored server.py while an unanchored rsync exclude silently ate skills/, because the verifier only looked for what had been missing. The structural twin: the husk shipped because the packaging pipeline was an untested code path with real users — "mass accumulates where mechanical enforcement can't see" applies to release engineering exactly as it applies to modules, and distribution is part of the product.
