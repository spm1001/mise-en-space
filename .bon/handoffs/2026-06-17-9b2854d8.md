## For Claudes to come

**The embedded/slim-flavour pattern (mise-hibere).** When a tool must be both a full desktop app and a lean embedded engine (Cornichon vendors mise), make the heavy deps an *optional extra*, not core — and carry the "full" choice in the *spawn* command, not the install command. Concretely: `markitdown[pdf]` + `pdf2image` live in an `extraction` extra; the plugin's mcpServers args spawn `uv run --extra extraction`, so the marketplace stays full automatically (the assembler vendors plugin.json), while an embedding consumer that installs plain core gets slim. You can't make an extra on-by-default, so "default install = slim, full opts in" is forced — but routing the opt-in through plugin.json means zero estate-wide install-command churn.

The unlock that made "slim" cheap rather than crippling: **profile what the weight actually buys.** The 67M monster was onnxruntime, pulled by magika — markitdown's ML *byte-level file-type sniffer*. Mise never needs it: it routes by Drive MIME metadata, not by sniffing bytes. And the two markitdown call sites already had graceful fallbacks (PDF → Drive server-side conversion; HTML → tag-stripping), Office was always Drive-based, and images only need pillow (14M, kept core). So dropping markitdown sheds ~84M and costs only fast-local-PDF (→ slower Drive round-trip) and PDF thumbnails. Lesson: before declaring a dependency load-bearing, check whether the expensive part answers a question you've already answered another way.

**The portmanteau marketplace is a shared single point of failure.** `spm1001/batterie` is the flat marketplace (install target) AND the assembler; it vendors 6 plugins from their source repos. The version ratchet (content-changed-without-bump → build FAIL) is a good guard, but its blanket `exit 1` means ONE repo's missed bump wedges the WHOLE bus — every plugin stops publishing. This bit live on 2026-06-17 (bon's unbumped Cowork docs blocked mise 0.7.9 + all). Two durable facts: (1) when a marketplace publish fails, check whether a SIBLING repo is the cause before hunting in your own; (2) the fix (bds-pujaki) is to quarantine the laggard to its last-good version and publish the rest, then go red to flag it — isolate the blast radius without a silent skip. Naming collision to hold: the *repo* `batterie` is the marketplace; the *plugin* `batterie` (suite utilities) is sourced from the *repo* `batterie-de-savoir`. Same word, two things.
