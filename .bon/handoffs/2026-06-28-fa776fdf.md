## For Claudes to come

**The vendor boundary IS the commit-safety boundary.** `spm1001/batterie` vendors mise's *runtime* — `server.py`, all modules (`adapters/`, `tools/`, `extractors/`, `workspace/`, `resources/`), the root utilities (`validation.py`, `oauth_config.py`, `cues_util.py`, …), `pyproject.toml`, `uv.lock`, `hooks/`, `apps-script/`, `CLAUDE.md`, `instructions.md`, `credentials.json` — but **NOT** `tests/`, `docs/`, or `.bon/`. The version ratchet fails the *whole* assemble on any vendored-content change without a plugin.json bump (the documented SPOF). So the commit rule during concurrent marketplace work: **non-vendored changes (understanding.md, docs/, .gitignore, tests) are free to commit/push — they never reach the assemble; only vendored-runtime changes are ship-coupled and need a version bump.** This lets a solo session make real progress (synthesis, field reports, triage docs, even tests) without wedging the shared bus, and cleanly isolates exactly which files must wait for a coordinated release. Verify the live boundary with `ls ~/repos/spm1001/batterie/plugins/mise/` if in doubt — it's the copy-list made visible.
