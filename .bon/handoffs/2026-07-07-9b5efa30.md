## For Claudes to come

**An MCP tool's real contract surface is its function signature — description text promising a parameter is not the parameter.** FastMCP generates the JSON schema from the do() signature in server.py; pydantic validates against that schema and silently DISCARDS any argument it doesn't declare (no error, no log). So a param can be documented in DO_DESCRIPTION_FULL, consumed by dispatch (`p.get("force", False)` fail-opens forever), recommended by the tool's own error messages — and never arrive. `force` lived in that state for two months. When adding any do() param: signature (server.py) + params dict + call_params log list + dispatch, and the drift-guard test in test_dispatch.py now enforces the signature half mechanically. Corollary for testing: test tool behaviour from the layer callers actually enter through, not the layer that's convenient to import.

**The setup_oauth flow contract (post-1.3.1): whoever mints the URL owns the PKCE verifier.** get_auth_url persists the verifier next to TOKEN_FILE; `python -m auth --auto --url <url>` is the listener half (opens browser best-effort, threaded listener on :3000, exchanges via jeton's persisted-verifier path). Never add a second mint anywhere in the flow — that was the original race. The verifier survives listener timeout and failed exchanges by design, so --code always works as the fallback. Three listener subtleties bought with incidents: single-threaded http.server wedges on Chrome's speculative second connection (use ThreadingHTTPServer); bind BEFORE webbrowser.open (a busy port must not burn a consent click); and port pre-checks must bind with SO_REUSEADDR to match http.server's own semantics, else TIME_WAIT ghosts from a just-finished flow fail the check for ~60s while the real listener would bind fine.

**The full remote OAuth dance is proven end-to-end on this estate** (2026-07-07, ~21:44): headless tube session → URL couriered via ~/scratch (Syncthing beats copy-paste for long URLs) → Mac browser click with `ssh -L 3000:localhost:3000 tube` up → callback through the tunnel → token verified against live Drive. This is the reference procedure for any headless re-auth; it needs a FRESH session after a mise ship (the MCP server loads code at session start — a session predating the ship runs the old server).
