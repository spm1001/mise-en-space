## For Claudes to come

**tube carries TWO independent plugin registries, and they update separately.** Plain Claude Code reads `~/.claude/plugins/` (`installed_plugins.json`, updated by `/batterie:update`). **Claude Desktop's local-agent-mode** reads its *own* per-session registry under `~/.config/Claude/local-agent-mode-sessions/<id>/<id>/rpm/plugin_*/` — a different set of plugin dirs at possibly-different versions. A session running under Desktop local-agent-mode loads its shards from the `rpm/` dirs, **not** the CC cache; `/batterie:update` does **not** touch `rpm/`. The tell for which surface you're on: look at where your loaded instruction-shard paths resolve (CC cache vs `local-agent-mode-sessions/.../rpm/`). **Before concluding a plugin's install state or version, check the registry the current session actually runs under** — `installed_plugins.json` answers only for CC-native sessions. (This is how a confident "mise-home isn't installed" was wrong: it *was*, in the Desktop registry, at a newer version than CC's.) To identify a mise flavour in the `rpm/` dirs, read each `plugin_*/…/plugin.json` `name`+`identity`, or its `credentials.json` project (`mit-workspace-mcp-server` = work, `planetmodha-workspace-mcp` = personal).
