## For Claudes to come

**A Dependabot merge to a vendored-full-source plugin (mise) arms a ratchet landmine — publish to clear it.** mise vendors its `pyproject.toml` + `uv.lock` into the marketplace (they "ride along" in `assemble.sh`'s full-source branch; only `tests/docs/fixtures/.bon` are excluded). So when a Dependabot PR auto-merges a dep bump to `main`, the shipped/vendored deps are now *behind* main. The next `assemble.yml` run (the 07:00 UTC daily cron, or any publish) re-vendors mise, detects changed content at an unchanged suite version, and **quarantines mise** (goes red — bds-pujaki isolates the blast radius but flags it). This bit tonight: PR #17 merged at 21:19 UTC; a *sibling* bon publish (1.9.1) had assembled 13 min earlier, so it re-stamped mise to 1.9.1 with the OLD deps — leaving main's pytest 9.1.1 orphaned. Cleared with a patch publish (1.9.2). **The rule: after a Dependabot PR merges to mise (or any full-source plugin), publish a patch to re-vendor and clear the ratchet — don't leave it for the cron.** The tell before it bites: shipped `plugins/mise/pyproject.toml` diverges from `main`'s. This composes with the existing "shared bus is a single point of failure" note above — same ratchet, different trigger (a dep merge rather than an unbumped content edit).

**Corollary — CLAUDE.md and any vendored source edit are ship-coupled the same way.** Committing a CLAUDE.md/source change to main without a publish arms the identical landmine. The estate convention (vajebi/winova) is therefore "don't docs-only-publish — bundle doc edits with the next *code* change." When there's no pending code change but a doc fix is wanted *now*, the elegant move is to *create* a small real code improvement to bundle it with (tonight: the gate hardening carried the CLAUDE.md refresh), turning a wasteful publish into a quality one.

**Design taste — a gate should key on its own state, not a proxy.** `configure_call_logging`'s `if _calls_logger.handlers: return` conflated "any handler exists" with "our file handler is wired." That's a fail-open proxy: it happened to work until a host (pytest 9.1) injected foreign handlers onto the logger. Keying on `isinstance(h, RotatingFileHandler)` is equivalent in production but honest about intent and robust to any handler-injecting harness. General shape: when an idempotency/"already done" check tests a proxy for the real condition, prefer checking the real condition.
